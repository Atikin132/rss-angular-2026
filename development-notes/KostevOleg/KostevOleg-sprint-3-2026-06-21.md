# Sprint 24 Cart feature - 2026-06-21

Сегодня завершил end-to-end интеграцию корзины с commercetools API.

## Что было сделано

* Полностью заменил локальный `CartStore` на `CartService`, работающий с commercetools API.
* Реализована загрузка активной корзины через `/me/active-cart`.
* Реализовано создание новой корзины при отсутствии активной.
* Добавлено добавление товаров в корзину.
* Реализовано изменение количества товаров.
* Реализовано удаление товаров из корзины.
* Реализована полная очистка корзины.

Для удаления товара из каталога был реализован отдельный flow поиска `lineItemId` по `productId`, так как commercetools удаляет позиции корзины только по `lineItemId`. Для этого был добавлен метод `removeFromCartByProductId`.

Вынес получение customer token в   `getCustomerToken(requiredScope)` и использовал его для profile и cart. Для корзины используется scope `manage_my_orders`, для профиля — `manage_my_profile`.

Добавил общий `PricePipe` на базе и подключил его во всех компонентах корзины вместо ручного форматирования денежных значений.

Реализовал поддержку промокодов:

* добавлен метод `applyPromoCode`;
* поле ввода промокода связано с `CartSummaryComponent` через Angular Signals;
* разделены `subtotalPrice` и `totalPrice`, чтобы отображать стоимость товаров до скидки и итоговую стоимость после применения промокода.

Исправил логику очистки корзины. Было обнаружено, что `clearCart()` удалял только товары, но оставлял применённый discount code. Из-за этого после повторного добавления товаров скидка автоматически применялась снова. Добавил отправку `removeDiscountCode`, благодаря чему очистка корзины теперь удаляет как товары, так и применённые промокоды.

## Архитектура

Во время реализации пришлось разобраться с особенностями  commercetools.

Корзина хранит товары как `lineItems`, а не как список продуктов. Большинство операций выполняется через `lineItemId`, хотя из каталога доступен только `productId`. Поэтому потребовалось реализовать преобразования между сущностями каталога и корзины.

Также итоговые суммы не рассчитываются на фронтенде. После каждого изменения корзины сервер возвращает полностью пересчитанную корзину, включая итоговую стоимость, применённые скидки и промокоды. 

## Рефакторинг

* Вынес получение customer token в общий helper, чтобы исключить дублирование логики получения токена с различными scopes.
* Добавил общий `PricePipe`, который теперь используется во всех компонентах корзины для единообразного форматирования цен.
* запровайдил через кастомный DI токе и useValue в верхнем инжекторе  параметры отображения волюты 

## Рефлексия
Во время реализации  корзины столкнулся с ошибкой архитектурного уровня: сначала вызов ensureCart() был размещён внутри CartButton. Так как CartButton рендерится в каждой карточке товара каталога, это приводило к множественным запросам GET /me/active-cart при открытии
страницы. Решение было перенести загрузку корзины на уровень родительских страниц (CatalogPage, ProductDetailsPage, CartPage), а CartButton оставить только для пользовательского действия: добавить или удалить товар. Это помогло лучше понять, где должна жить загрузка данных, а где только UI interaction.

Отдельно понравился подход с InjectionToken для настроек форматирования цены. Вместо того чтобы держать дефолтную валюту и локаль прямо внутри PricePipe,  вынес эти значения в конфиг приложения и запровайдили их через корневой DI. Это делает pipe более универсальным: он больше не привязан жёстко к USD и en-US, а использует настройки приложения по умолчанию. Если позже потребуется сменить валюту или локаль для всего проекта, достаточно изменить provider в одном месте, не трогая сам pipe и шаблоны.

Основная сложность оказалась связана не с Angular, а с пониманием модели commercetools. Пришлось разобраться, как работает cart update flow, зачем используется `version`, чем отличаются `productId` и `lineItemId`, каким образом применяются промокоды и почему итоговую стоимость необходимо брать из ответа сервера, а не рассчитывать самостоятельно.

Дополнительно возникла проблема с данными проекта: один из товаров нельзя было добавить в корзину из-за отсутствующей `Tax Category`. Это помогло понять, что часть ошибок может находиться не во фронтенде, а в конфигурации commercetools.
