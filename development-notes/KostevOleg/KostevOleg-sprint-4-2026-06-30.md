# Sprint 4 Tests for About, Cart and Profile - 2026-06-30


Сегодня работал с unit-тестами для **About Page**, **корзины** и **профиля пользователя**. Основной задачей было разобраться какие подходы используются при тестировании компонентов и сервисов.

Для **About Page** были проверены тесты, которые подтверждают создание страницы и корректное отображение основного контента: заголовка, описания проекта, карточек участников команды, практик совместной работы и списка используемых инструментов.

Для **корзины** были изучены тесты сервиса и компонентов. Проверяются загрузка активной корзины, создание новой корзины при её отсутствии, добавление и удаление товаров, изменение количества, применение промокода, очистка корзины, а также события компонентов `CartItem`, `CartList`, `CartSummary` и `CartBadge`.

Для **профиля** были рассмотрены тесты страницы профиля, сервисов и отдельных компонентов. Покрываются загрузка пользователя, редактирование личных данных, валидация форм, сохранение изменений, отмена редактирования, смена пароля, работа с адресами и обработка ошибок при обновлении данных.

Во время работы также был выполнен общий запуск тестов командой. Проверенные тесты работают корректно.


### Рефлексия

Главным выводом стало то, что хорошая архитектура напрямую упрощает тестирование. Компоненты и сервисы с одной чёткой ответственностью значительно легче покрывать тестами и поддерживать в дальнейшем.

Также было полезно разобраться с тестированием Angular standalone components, использованием `TestBed`, моками сервисов и проверкой взаимодействия с API без выполнения реальных HTTP-запросов. Теперь гораздо лучше понимаю, как строятся unit-тесты как для компонентов, так и для бизнес-логики приложения.
